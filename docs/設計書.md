# 投資管理アプリ 設計書

最終更新: 2026-05-30

---

## 1. プロジェクト概要

個人投資家向けのポートフォリオ管理Webアプリ。AI（Ollama / ローカルLLM）による投資アドバイスと、東証全銘柄のスクリーニング機能を備える。

| 項目 | 内容 |
|------|------|
| プロジェクト名 | investment-app |
| フレームワーク | Next.js 16.2.6 (App Router, Turbopack) |
| 言語 | TypeScript |
| スタイル | Tailwind CSS |
| DB | PostgreSQL + Prisma ORM (v7) |
| AIエンジン | Ollama (llama3.2, ローカル実行) |
| 株価データ | Yahoo Finance (yahoo-finance2) |
| チャート | Recharts |

## 3. データベーススキーマ

### Holding（保有銘柄）
ユーザーが保有している銘柄を管理する。

| カラム | 型 | 説明 |
|--------|-----|------|
| id | String (cuid) | PK |
| name | String | 銘柄名 |
| ticker | String | ティッカーシンボル |
| quantity | Float | 保有数量 |
| purchasePrice | Float | 購入単価 |
| currentPrice | Float | 現在値 |
| sector | String | セクター |
| per | Float? | PER |
| pbr | Float? | PBR |
| dividendYield | Float? | 配当利回り(%) |
| marketCap | Float? | 時価総額 |
| createdAt | DateTime | 作成日時 |
| updatedAt | DateTime | 更新日時 |

### Stock（東証全銘柄マスタ）
Yahoo Financeから取得した東証上場銘柄の情報を格納する。スクリーニングのベースデータ。

| カラム | 型 | 説明 |
|--------|-----|------|
| id | String (cuid) | PK |
| ticker | String (unique) | ティッカー (例: 7203.T) |
| name | String | 銘柄名 |
| sector | String? | セクター |
| market | String? | 市場区分 (プライム/スタンダード/グロース) |
| currentPrice | Float? | 現在値 |
| per | Float? | PER |
| pbr | Float? | PBR |
| dividendYield | Float? | 配当利回り(%) |
| marketCap | Float? | 時価総額 |
| eps | Float? | EPS |
| roe | Float? | ROE(%) |
| lastUpdated | DateTime? | Yahoo Finance最終取得日時 |
| createdAt | DateTime | 作成日時 |
| updatedAt | DateTime | 更新日時 |

### SyncJob（バッチ同期ジョブ）
Yahoo Financeバッチ取得の進捗を管理する。

| カラム | 型 | 説明 |
|--------|-----|------|
| id | String (cuid) | PK |
| status | String | pending / running / completed / failed |
| total | Int | 対象銘柄数 |
| processed | Int | 処理済み件数 |
| failed | Int | 失敗件数 |
| startedAt | DateTime | 開始日時 |
| completedAt | DateTime? | 完了日時 |

## 4. API設計

### 保有銘柄 API

| メソッド | パス | 説明 |
|----------|------|------|
| GET | `/api/holdings?q=&sector=` | 一覧取得（部分検索・セクター絞込） |
| POST | `/api/holdings` | 新規追加 |
| PUT | `/api/holdings/[id]` | 更新 |
| DELETE | `/api/holdings/[id]` | 削除 |

### 株価データ API

| メソッド | パス | 説明 |
|----------|------|------|
| GET | `/api/stock/search?q=` | 銘柄名・ティッカー部分検索 (Yahoo Finance) |
| GET | `/api/stock/quote/[ticker]` | 個別銘柄の最新価格・指標取得 |

### 東証全銘柄 API（スクリーニング用）

| メソッド | パス | 説明 |
|----------|------|------|
| GET | `/api/stocks?q=&sector=&perMax=&pbrMax=&dyMin=&marketCapMin=&limit=50` | スクリーニング検索 |
| POST | `/api/stocks/seed` | 日経225の初期データをDBに登録 |
| POST | `/api/stocks/sync` | Yahoo Financeから財務指標をバッチ取得 (10件/回) |
| POST | `/api/stocks/import` | JPX CSVファイルを取り込みDBに一括登録 |

### AIアドバイス API

| メソッド | パス | 説明 |
|----------|------|------|
| POST | `/api/ai-advice` | Ollamaにストリーミングリクエスト（保有銘柄のコンテキスト付き） |

## 5. スクリーニング機能 設計

### 概要
東証全銘柄（最大約3,800銘柄）を対象に、複数の財務指標で絞り込むスクリーニング機能。

### データ取得フロー
```
① 初回セットアップ
   JPX CSV取り込み or 日経225シード
        ↓
   Stock テーブルに銘柄名・ティッカーを登録
        ↓
② バッチ同期（手動トリガー or 定期実行）
   POST /api/stocks/sync を繰り返し呼び出し
        ↓
   10件ずつYahoo Financeから取得
   (200ms/件 のレートリミット)
        ↓
   Stock テーブルに財務指標を更新
        ↓
③ スクリーニング実行
   GET /api/stocks?perMax=15&dyMin=2...
        ↓
   Stock テーブルをクエリして結果返却
```

### レートリミット対策
- 1件取得ごとに200ms待機（= 約5件/秒）
- 1バッチ10件 → 約2秒/バッチ
- 日経225（225件）: 約45秒で完了
- 東証全銘柄（3,800件）: 約12分で完了
- 失敗した銘柄はスキップ（SyncJobのfailedカウントに記録）
- `lastUpdated`が24時間以内の銘柄は再取得しない

### スクリーニング条件
| 条件 | パラメータ | 説明 |
|------|-----------|------|
| キーワード | `q` | 銘柄名・ティッカー部分一致 |
| セクター | `sector` | 業種絞込 |
| PER上限 | `perMax` | 割安株の抽出 |
| PBR上限 | `pbrMax` | 資産割安株の抽出 |
| 配当利回り下限 | `dyMin` | 高配当株の抽出 |
| 時価総額下限 | `marketCapMin` | 大型株に絞込 |
| 時価総額上限 | `marketCapMax` | 小型株に絞込 |

### JPX CSVフォーマット（取り込み対応）
JPX（日本取引所グループ）が公開している上場銘柄一覧CSV:
- URL: https://www.jpx.co.jp/markets/statistics-equities/misc/01.html
- ダウンロードボタン: 「上場銘柄一覧（Excel）」
- 主要カラム: コード, 銘柄名, 市場区分, 33業種区分
